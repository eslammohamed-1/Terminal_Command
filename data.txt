# 1. عقلية التيرمنال وتجهيز البيئة

## 1.1 ما هو Terminal؟ وما هو Shell؟

الـ **Terminal** هو النافذة التي تكتب فيها الأوامر.

الـ **Shell** هو البرنامج الذي يقرأ الأمر ويفهمه وينفذه.

أمثلة على الـ Shell:

```bash
bash
zsh
sh
```

على Linux غالبًا ستجد Bash.  
على macOS الحديث غالبًا الافتراضي هو Zsh.

لمعرفة الـ shell الحالي:

```bash
echo "$SHELL"
```

لمعرفة اسم النظام:

```bash
uname -s
```

الناتج غالبًا:

```text
Linux
```

أو:

```text
Darwin
```

`Darwin` يعني macOS.

***

## 1.2 الفرق بين Linux وmacOS في التيرمنال

Linux غالبًا يستخدم أدوات GNU.

macOS يستخدم أدوات BSD افتراضيًا.

الأمر قد يكون بنفس الاسم، مثل:

```bash
sed
date
grep
find
```

لكن بعض الخيارات تختلف بين النظامين.

الفكرة المهمة:

> الأوامر الأساسية غالبًا مشتركة، لكن الأوامر المتقدمة قد تختلف في التفاصيل.

***

## 1.3 جعل البيئتين متشابهتين قدر الإمكان

### على macOS ثبت Homebrew

Homebrew هو مدير حزم شائع على macOS.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

بعد التثبيت:

```bash
brew doctor
brew --version
```

### تثبيت أدوات GNU على macOS

```bash
brew install coreutils findutils gnu-sed gawk grep
```

بعدها ستجد نسخ GNU بأسماء مثل:

```bash
gsed
gawk
ggrep
gdate
```

مثال:

```bash
gdate --version
gsed --version
```

# 2. فهم شكل الأوامر والمسارات

## 2.1 شكل الأمر العام

معظم الأوامر شكلها هكذا:

```bash
command [options] [arguments]
```

مثال:

```bash
ls -lah /tmp
```

المعنى:

```text
ls       الأمر
-lah     خيارات
/tmp     المسار أو الهدف
```

بعض الخيارات قصيرة:

```bash
ls -l
```

وبعضها طويلة:

```bash
ls --help
```

أحيانًا يمكن دمج الخيارات القصيرة:

```bash
ls -l -a -h
```

نفس:

```bash
ls -lah
```

***

## 2.2 المسارات Paths

### المسار المطلق Absolute Path

يبدأ من جذر النظام `/`.

على Linux:

```bash
/home/ahmed/projects
```

على macOS:

```bash
/Users/ahmed/projects
```

### المسار النسبي Relative Path

يبدأ من مكانك الحالي.

```bash
./app
../backup
docs/file.txt
```

رموز مهمة:

```text
/    جذر النظام
~    مجلد المستخدم الحالي
.    المجلد الحالي
..   المجلد الأب
```

أمثلة:

```bash
cd ~
cd ..
cd ./src
cd /tmp
```

***

## 2.3 كيف تطلب المساعدة؟

### man

```bash
man ls
```

داخل صفحة `man`:

```text
q       خروج
/word   بحث
n       النتيجة التالية
```

### help

بعض الأوامر داخلية في الـ shell مثل `cd`.

```bash
help cd
```

### type

يعرفك الأمر ده نوعه إيه ومكانه فين.

```bash
type cd
type ls
type python3
```

مثال ناتج:

```text
cd is a shell builtin
ls is /bin/ls
```

### which و command -v

```bash
which git
```

أو الأفضل في السكربتات:

```bash
command -v git
```

# 3. أوامر الملفات والمجلدات

## 3.1 pwd — اعرف مكانك الحالي

### الوظيفة

يعرض المسار الحالي.

```bash
pwd
```

مثال ناتج:

```text
/home/ahmed/projects
```

أو:

```text
/Users/ahmed/projects
```

***

## 3.2 ls — عرض الملفات والمجلدات

### الوظيفة

يعرض محتوى مجلد.

```bash
ls
```

عرض تفصيلي:

```bash
ls -l
```

عرض المخفي:

```bash
ls -a
```

عرض تفصيلي ومخفي وبأحجام مفهومة:

```bash
ls -lah
```

### فهم الناتج

```text
-rw-r--r--  1 ahmed staff  1200 Jun 4 notes.txt
```

المعنى:

```text
-rw-r--r--   الصلاحيات
1            عدد الروابط
ahmed        المالك
staff        المجموعة
1200         الحجم
Jun 4        آخر تعديل
notes.txt    اسم الملف
```

### خطأ شائع

لو اسم الملف فيه مسافة:

```bash
ls my file.txt
```

ده غلط لأنه سيفهمها كملفين.

الصحيح:

```bash
ls "my file.txt"
```

أو:

```bash
ls my\ file.txt
```

***

## 3.3 cd — التنقل بين المجلدات

الذهاب لمجلد:

```bash
cd Documents
```

الذهاب للبيت:

```bash
cd ~
```

الصعود مستوى:

```bash
cd ..
```

الرجوع للمكان السابق:

```bash
cd -
```

مثال:

```bash
cd ~/Documents
pwd
```

***

## 3.4 mkdir — إنشاء مجلد

إنشاء مجلد:

```bash
mkdir project
```

إنشاء أكثر من مجلد:

```bash
mkdir src tests docs
```

إنشاء مسار كامل:

```bash
mkdir -p project/src/components
```

`-p` معناها: أنشئ المجلدات الناقصة ولا تعطيني خطأ لو موجودة.

***

## 3.5 touch — إنشاء ملف فارغ أو تحديث وقته

```bash
touch notes.txt
```

إنشاء عدة ملفات:

```bash
touch a.txt b.txt c.txt
```

***

## 3.6 cp — نسخ

نسخ ملف:

```bash
cp file.txt copy.txt
```

نسخ ملف إلى مجلد:

```bash
cp file.txt backup/
```

نسخ مجلد:

```bash
cp -r project project-backup
```

`-r` معناها recursive، أي انسخ كل ما داخل المجلد.

***

## 3.7 mv — نقل أو إعادة تسمية

إعادة تسمية:

```bash
mv old.txt new.txt
```

نقل ملف:

```bash
mv file.txt ~/Documents/
```

نقل مجلد:

```bash
mv project ~/Desktop/
```

***

## 3.8 rm — حذف

حذف ملف:

```bash
rm file.txt
```

حذف مجلد ومحتواه:

```bash
rm -r folder
```

حذف بالقوة:

```bash
rm -rf folder
```

> تحذير مهم:  
> `rm -rf` لا يرسل الملفات إلى سلة المهملات. استخدمه بحذر.

قبل الحذف يمكنك تجربة:

```bash
ls folder
```

أو:

```bash
find folder
```

لكي ترى ما سيتم حذفه.

***

## 3.9 file — معرفة نوع الملف

```bash
file image.png
file script.sh
file archive.tar.gz
```

مثال ناتج:

```text
script.sh: Bourne-Again shell script text executable
```

## 3.10 stat — معلومات تفصيلية عن الملف

على Linux:

```bash
stat file.txt
```

على macOS:


لو تريد تجنب الاختلافات في السكربتات، استخدم أوامر أبسط مثل:

```bash
ls -l file.txt
```

أو Python عند الحاجة.

# 4. الصلاحيات والمستخدمين

## 4.1 فهم الصلاحيات

نفذ:

```bash
ls -l
```

مثال:

```text
-rwxr-xr--  1 ahmed staff  1200 Jun 4 script.sh
```

أول جزء:

```text
-rwxr-xr--
```

ينقسم إلى:

```text
-     نوع الملف
rwx   صلاحيات المالك
r-x   صلاحيات المجموعة
r--   صلاحيات الآخرين
```

الحروف:

```text
r = read    قراءة
w = write   كتابة
x = execute تنفيذ
```

***

## 4.2 chmod — تغيير الصلاحيات

جعل سكربت قابل للتنفيذ:

```bash
chmod +x script.sh
```

إزالة التنفيذ:

```bash
chmod -x script.sh
```

استخدام الأرقام:

```bash
chmod 755 script.sh
```

المعنى:

```text
7 = read + write + execute
5 = read + execute
5 = read + execute
```

أشهر القيم:

```text
644  ملف عادي
755  سكربت أو برنامج قابل للتنفيذ
700  خاص بالمالك فقط
```

***

## 4.3 chown — تغيير المالك

تغيير مالك ملف:

```bash
sudo chown ahmed file.txt
```

تغيير المالك والمجموعة:

```bash
sudo chown ahmed:staff file.txt
```

تغيير مجلد وكل محتواه:

```bash
sudo chown -R ahmed:staff project
```

> انتبه:  
> `chown -R` خطير إذا استخدمته على مسار غلط.

***

## 4.4 sudo — تنفيذ كمدير

```bash
sudo command
```

مثال Linux:

```bash
sudo apt update
```

مثال macOS:

```bash
sudo softwareupdate -l
```

قاعدة مهمة:

> لا تستخدم `sudo` لمجرد أن الأمر فشل. اقرأ رسالة الخطأ أولًا.

***

## 4.5 whoami و id

معرفة المستخدم الحالي:

```bash
whoami
```

معرفة المستخدم والمجموعات:

```bash
id
```

# 5. التعامل مع النصوص والبيانات

## 5.1 cat — عرض محتوى ملف

```bash
cat file.txt
```

مناسب للملفات الصغيرة.

لإنشاء ملف سريعًا:

```bash
cat > notes.txt
```

اكتب النص، ثم اضغط:

```text
Ctrl + D
```

***

## 5.2 less — قراءة ملف كبير

```bash
less app.log
```

داخل less:

```text
Space  صفحة للأسفل
b      صفحة للأعلى
/word  بحث
n      التالي
q      خروج
```

***

## 5.3 head — أول أسطر

```bash
head file.txt
```

أول 20 سطر:

```bash
head -n 20 file.txt
```

***

## 5.4 tail — آخر أسطر

```bash
tail file.txt
```

آخر 50 سطر:

```bash
tail -n 50 file.txt
```

متابعة ملف لحظيًا:

```bash
tail -f app.log
```

مفيد جدًا في logs.

***

## 5.5 grep — البحث داخل النصوص

بحث عادي:

```bash
grep "error" app.log
```

بدون حساسية لحالة الأحرف:

```bash
grep -i "error" app.log
```

مع رقم السطر:

```bash
grep -n "error" app.log
```

بحث recursively داخل مجلد:

```bash
grep -R "TODO" .
```

بحث مع تجاهل case ورقم السطر:

```bash
grep -Rni "TODO" .
```

### grep -q في السكربتات

```bash
if grep -q "ERROR" app.log; then
  echo "Errors found"
else
  echo "No errors"
fi
```

`-q` معناها quiet، لا تطبع الناتج، فقط اعتمد على كود النجاح أو الفشل.

***

## 5.6 wc — العد

```bash
wc file.txt
```

يعرض:

```text
lines words bytes
```

عدد السطور:

```bash
wc -l file.txt
```

مثال عملي:

```bash
grep "ERROR" app.log | wc -l
```

***

## 5.7 sort — الترتيب

```bash
sort names.txt
```

ترتيب عكسي:

```bash
sort -r names.txt
```

ترتيب رقمي:

```bash
sort -n numbers.txt
```

***

## 5.8 uniq — حذف التكرار المتجاور

```bash
uniq names.txt
```

لكن غالبًا تستخدمه مع `sort`:

```bash
sort names.txt | uniq
```

عدّ التكرارات:

```bash
sort names.txt | uniq -c
```

***

## 5.9 cut — استخراج أعمدة

مثال من `/etc/passwd`:

```bash
cut -d ':' -f 1 /etc/passwd
```

المعنى:

```text
-d ':'   الفاصل هو :
-f 1     العمود الأول
```

***

## 5.10 tr — تحويل أو حذف حروف

تحويل إلى uppercase:

```bash
echo "hello" | tr 'a-z' 'A-Z'
```

حذف المسافات:

```bash
echo "a b c" | tr -d ' '
```

# 6. Pipes و Redirection

## 6.1 Pipe

الـ pipe يأخذ output من أمر ويدخله input للأمر التالي.

```bash
command1 | command2
```

مثال:

```bash
grep "404" access.log | wc -l
```

المعنى:

1. ابحث عن السطور التي تحتوي `404`.
2. عدّ عددها.

مثال أقوى:

```bash
cat access.log | grep "ERROR" | sort | uniq -c
```

لكن الأفضل:

```bash
grep "ERROR" access.log | sort | uniq -c
```

لأن `grep` يستطيع قراءة الملف مباشرة.

***

## 6.2 Redirection

### كتابة الناتج في ملف

```bash
echo "hello" > file.txt
```

`>` يمسح القديم ويكتب من جديد.

### الإضافة على ملف

```bash
echo "world" >> file.txt
```

`>>` يضيف في آخر الملف.

### حفظ الأخطاء في ملف

```bash
ls not-found 2> errors.txt
```

### حفظ الناتج والأخطاء

```bash
command > output.txt 2> errors.txt
```

أو جمع الاثنين:

```bash
command > all.txt 2>&1
```

# 7. البحث عن الملفات ومعالجة النتائج

## 7.1 find — البحث عن الملفات

بحث باسم:

```bash
find . -name "*.log"
```

بحث عن ملفات فقط:

```bash
find . -type f -name "*.txt"
```

بحث عن مجلدات فقط:

```bash
find . -type d -name "node_modules"
```

بحث بدون حساسية لحالة الأحرف:

```bash
find . -iname "readme.md"
```

حذف ملفات log بحذر:

```bash
find . -type f -name "*.log" -print
```

بعد التأكد:

```bash
find . -type f -name "*.log" -delete
```

> ملاحظة:  
> خيارات `find` المتقدمة قد تختلف بين GNU وBSD، لذلك اختبر الأمر قبل استخدامه في سكربت مهم.

***

## 7.2 xargs — تنفيذ أمر على نتائج أمر آخر

مثال:

```bash
find . -name "*.log" | xargs rm
```

لكن لو أسماء الملفات فيها مسافات، استخدم:

```bash
find . -name "*.log" -print0 | xargs -0 rm
```

هذه الطريقة أكثر أمانًا.

# 8. العمليات والنظام

## 8.1 ps — عرض العمليات

```bash
ps aux
```

البحث عن عملية:

```bash
ps aux | grep nginx
```

لكن سيظهر أمر grep نفسه أحيانًا.

حل بسيط:

```bash
ps aux | grep "[n]ginx"
```

***

## 8.2 top و htop

عرض العمليات لحظيًا:

```bash
top
```

لو مثبت `htop`:

```bash
htop
```

تثبيته:

Ubuntu/Debian:

```bash
sudo apt install htop
```

macOS:

```bash
brew install htop
```

Arch:

```bash
sudo pacman -S htop
```

***

## 8.3 kill — إيقاف عملية

إيقاف عادي:

```bash
kill PID
```

مثال:

```bash
kill 1234
```

إيقاف قوي:

```bash
kill -9 1234
```

> `kill -9` آخر حل، لأنه لا يعطي البرنامج فرصة ينظف نفسه أو يحفظ بياناته.

***

## 8.4 jobs و fg و bg

تشغيل أمر في الخلفية:

```bash
sleep 100 &
```

عرض jobs:

```bash
jobs
```

إرجاع job للأمام:

```bash
fg
```

استكماله في الخلفية:

```bash
bg
```

إيقاف مؤقت لعملية في foreground:

```text
Ctrl + Z
```

إيقاف نهائي غالبًا:

```text
Ctrl + C
```

***

## 8.5 df و du — المساحة

مساحة الأقراص:

```bash
df -h
```

حجم مجلد:

```bash
du -sh project
```

أكبر عناصر داخل مجلد:

```bash
du -sh * | sort -h
```

> `sort -h` يعمل في GNU، وقد لا يتوفر بنفس الشكل في macOS القديم. لو واجهت مشكلة على macOS، استخدم `brew install coreutils` ثم `gsort`.

# 9. إدارة الحزم والبرامج

## 9.1 Ubuntu/Debian — apt

تحديث الفهارس:

```bash
sudo apt update
```

ترقية الحزم:

```bash
sudo apt upgrade
```

تثبيت برنامج:

```bash
sudo apt install nginx
```

حذف برنامج:

```bash
sudo apt remove nginx
```

حذف مع ملفات الإعداد:

```bash
sudo apt purge nginx
```

تنظيف الاعتمادات غير المستخدمة:

```bash
sudo apt autoremove
```

***

## 9.2 Arch Linux — pacman

تحديث النظام:

```bash
sudo pacman -Syu
```

تثبيت برنامج:

```bash
sudo pacman -S nginx
```

حذف برنامج:

```bash
sudo pacman -R nginx
```

حذف مع الاعتمادات غير المستخدمة:

```bash
sudo pacman -Rs nginx
```

البحث:

```bash
pacman -Ss nginx
```

معلومات عن حزمة:

```bash
pacman -Si nginx
```

***

## 9.3 macOS — Homebrew

تحديث Homebrew:

```bash
brew update
```

تثبيت أداة:

```bash
brew install wget
```

تثبيت تطبيق GUI:

```bash
brew install --cask firefox
```

ترقية البرامج:

```bash
brew upgrade
```

عرض المثبت:

```bash
brew list
```

حذف برنامج:

```bash
brew uninstall wget
```

تنظيف:

```bash
brew cleanup
```

# 10. GNU/Linux vs BSD/macOS

## 10.1 sed

استبدال نص وعرض الناتج فقط:

```bash
sed 's/old/new/g' file.txt
```

### التعديل داخل الملف

Linux GNU sed:

```bash
sed -i 's/old/new/g' file.txt
```

macOS BSD sed:

```bash
sed -i '' 's/old/new/g' file.txt
```

### طريقة آمنة تعمل على الاثنين

```bash
sed 's/old/new/g' file.txt > file.tmp && mv file.tmp file.txt
```

***

## 10.2 awk

عرض العمود الأول:

```bash
awk '{print $1}' file.txt
```

استخدام فاصل:

```bash
awk -F ':' '{print $1}' /etc/passwd
```

غالبًا الأساسيات تعمل على النظامين.

لو احتجت GNU awk على macOS:

```bash
brew install gawk
```

ثم:


### تاريخ الغد

## 11.11 سكربت portable كامل

```bash
#!/usr/bin/env bash
set -euo pipefail

log() {
  echo "[INFO] $*"
}

get_tomorrow() {
  case "$(uname -s)" in
    Linux)
      date -d "tomorrow" "+%Y-%m-%d"
      ;;
    Darwin)
      date -v+1d "+%Y-%m-%d"
      ;;
    *)
      python3 - <<'PY'
from datetime import datetime, timedelta
print((datetime.now() + timedelta(days=1)).strftime("%Y-%m-%d"))
PY
      ;;
  esac
}

main() {
  log "OS: $(uname -s)"
  log "User: $(whoami)"
  log "Tomorrow: $(get_tomorrow)"
}

main "$@"
```

# قواعد ذهبية لازم تحفظها بالفهم

1. اقرأ رسالة الخطأ قبل ما تستخدم `sudo`.
2. أي متغير في سكربت ضعه بين quotes:

```bash
"$var"
```

3. لا تستخدم `rm -rf` إلا وأنت متأكد من المسار.
4. استخدم `man`, `help`, `type` بدل الحفظ الأعمى.
5. افهم الفرق بين:

```bash
>
```

و:

```bash
>>
```

6. افهم أن `|` ينقل output من أمر إلى input لأمر آخر.
7. لو السكربت سيعمل على Linux وmacOS، احذر من:

```text
sed
date
stat
find
grep
```

8. استخدم:

```bash
uname -s
```

للتفريق بين الأنظمة.

9. في macOS، Homebrew صديقك.
10. في Linux، مدير الحزم يعتمد على التوزيعة.
11. لا تنشئ ملفات تختلف فقط في الحروف الكبيرة والصغيرة.
12. جرّب الأمر على ملفات اختبار قبل تشغيله على ملفات مهمة.

## الخلاصة

لو مشيت على هذا الكتيب بالترتيب، أنت مش هتحفظ أوامر فقط؛ هتفهم:

* التيرمنال بيفكر إزاي.
* النظام بيشوف الملفات إزاي.
* الأوامر بتاخد input وتطلع output إزاي.
